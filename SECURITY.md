# Security policy

WonderDrive accepts untrusted audience questions, retrieved web pages, model output, and source URLs. Treat all four as data—not instructions—and keep provider/database credentials out of the browser.

## Reporting a vulnerability

Please do not open a public GitHub issue for a suspected vulnerability or exposed credential. Contact the repository owner privately through the GitHub profile associated with this repository and include:

- the affected route or component;
- reproduction steps;
- the likely impact;
- any safe mitigation you have already tested.

Do not include real API keys, personal data, or exploit payloads that could harm third parties.

## Supported version

Only the latest commit on `main` is supported during the hackathon build.
